[[git]] предоставляет нам теги, чья основная задача – ссылаться постоянно на конкретный [[commit]]. Важно, что после создания они никогда не сменят своего положения.

``` bash
git tag <опция> <имя_тега> <commit>
```

|Опция|Назначение|
|---|---|
|`-a`|Создать аннотированный тег|
|`-m`|Добавить сообщение к аннотированному тегу|
|`-d`|Удалить тег|
|`-l`|Показать список тегов|
|`-n`|Показать сообщения тегов|
|`-f`|Перезаписать существующий тег|
``` bash
git tag v1.0.0 # Создать простой тег
git tag -a v1.0.0 -m "First release" # Создать аннотированный тег
git tag v1.0.0 a1b2c3 # Создать тег для определённого коммита
git tag -d v1.0.0 # Удалить тег
git tag # Показать список тегов
```

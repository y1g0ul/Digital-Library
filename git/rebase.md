При ребейзе [[git]] по сути копирует набор [[commit]]ов и переносит их в другое место. C его помощью можно делать чистые и красивые линейные последовательности [[commit]]ов.
``` bash
git rebase <опция> <ветка>
```

![[git rebase]]

| Опция        | Назначение                                                                                                                                                                      |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `-i`         | Опция для открытия интерфейса с целью просмора того какие [[commit]]ы готовы к копированию. Так же показывает хеши [[commit]]ов и комментарии к ним. ( `git rebase -i HEAD~4` ) |
| `--continue` | Продолжить rebase после разрешения конфликтов                                                                                                                                   |
| `--abort`    | Отменить rebase                                                                                                                                                                 |
| `--skip`     | Пропустить текущий коммит                                                                                                                                                       |
``` bash
git rebase main # Перенести текущую ветку поверх main
git rebase -i HEAD~3 # Изменить последние 3 коммита
git rebase --continue # Продолжить после конфликта
git rebase --abort # Отменить rebase
```

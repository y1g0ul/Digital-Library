Это простой способ в [[git]] копировать несколько [[commit]]ов на место, где сейчас находится [[git/HEAD|HEAD]].  Команда поместит любой коммит сразу после [[git/HEAD|HEAD]] (только если этот коммит не является предком [[git/HEAD|HEAD]]).

``` bash
git cherry-pick <опция> <commit>
```

| Опция               | Назначение                                        |
| ------------------- | ------------------------------------------------- |
| `-n`, `--no-commit` | Применить изменения без создания коммита          |
| `-e`                | Изменить сообщение коммита перед созданием        |
| `--continue`        | Продолжить после разрешения конфликта             |
| `--abort`           | Отменить `cherry-pick` при конфликте              |
| `--skip`            | Пропустить текущий коммит при серии `cherry-pick` |
``` bash
git cherry-pick a1b2c3 # Применить один коммит
git cherry-pick a1b2c3 d4e5f6 # Применить несколько коммитов
git cherry-pick -n a1b2c3 # Применить без создания коммита
git cherry-pick --abort # Отменить операцию
git cherry-pick --continue # Продолжить после разрешения конфликта
```

Так же можно перемещать несколько изменений с помощью [[rebase]] и опции `-i`.

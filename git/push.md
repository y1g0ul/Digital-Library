Команда в [[git]] отвечающая за загрузку ваших изменений в указанный репозиторий, а также включение ваших [[commit]]ов в состав удаленного репозитория.
``` bash
git push <опция> <remote> <ветка>
```

| Опция                | Назначение                                     |
| -------------------- | ---------------------------------------------- |
| `-u`                 | Связать локальную ветку с удалённой (upstream) |
| `--force`            | Принудительно перезаписать удалённую историю   |
| `--force-with-lease` | Безопасный force push с проверкой изменений    |
| `--all`              | Отправить все локальные ветки                  |
| `--tags`             | Отправить все теги                             |
| `--delete`           | Удалить удалённую ветку                        |
> [!warning] Предупреждение
> Поведение команды `git push` без аргументов варьируется в зависимости от значения `push.default`, указанной в настройках [[git]]. Значение по умолчанию зависит от версии git, которую вы используете. Лучше всегда проверять эту опцию прежде чем push-ить ваши настоящие проекты.

Команда может быть использована и с аргументами - `git push <удалённый_репозиторий> <целевая_ветка>` 

В том случае, когда вам необходимо разделить источник и получатель аргумента - `git push origin <источник>:<получатель>`

Если мы не укажем источник `git push origin :<получатель>` то [[branch]] получателя будет удалена. 
``` bash
git push # Отправить изменения в связанную ветку
git push origin main # Отправить ветку main в origin
git push -u origin main # Отправить и установить upstream
git push --tags # Отправить все теги
git push origin --delete feature # Удалить удалённую ветку
```

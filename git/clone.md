`git clone`  - это команда в [[git]], которая создаст локальную копию удалённого репозитория.
``` bash
git clone <опция> <URL> <директория>
```

| Опция                  | Назначение                                                    |
| ---------------------- | ------------------------------------------------------------- |
| `--branch <ветка>`     | Клонировать указанную ветку                                   |
| `--depth <n>`          | Выполнить поверхностное клонирование (последние `n` коммитов) |
| `--single-branch`      | Клонировать только одну ветку                                 |
| `--recurse-submodules` | Клонировать репозиторий вместе с подмодулями                  |
После `git clone` [[git]] автоматически:
- создаёт локальный репозиторий;
- добавляет удалённый репозиторий с именем `origin`;
- переключается на основную [[branch]] (`main` или `master`);
- загружает всю историю [[commit]]ов (если не указан `--depth`).

`<удалённый репозиторий>/<имя ветки>` - В начале названия используется в удаленных [[branch]].( `origin/main`, то здесь `main` - это имя ветки, а `origin` - это имя удалённого репозитория. ). Эти ветки отображаются на локальном репозитории как ссылки на состояние [[branch]] удаленного репозитория.

Удаленные ветки обновятся тогда и только тогда, когда обновится сам удалённый репозиторий.

``` bash
git clone https://github.com/user/repo.git # Клонировать репозиторий
git clone https://github.com/user/repo.git my-project # Клонировать в папку my-project
git clone --branch develop https://github.com/user/repo.git # Клонировать ветку develop
git clone --depth 1 https://github.com/user/repo.git # Скачать только последний коммит
git clone --recurse-submodules https://github.com/user/repo.git # Клонировать вместе с подмодулями
```

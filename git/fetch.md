Команда в [[git]] которая выполняет только 2 основные опперации:
- связывается с удаленным репозиторием и забирает все те данные проекта, которых у вас нет
- у вас должны появиться ссылки на все [[branch]] из этого удаленного репозитория (например, `o/main`)

Фактически `git fetch` синхронизирует локальное представление удаленных репозиториев с тем, что является актуальным на текущий момент времени. 

> [!warning] Важно отметить
> Команда всего лишь скачивает все необходимые данные, но вам потребуется вручную слить эти данные с вашими.

``` bash
git fetch <опция> <remote>
```

| Опция       | Назначение                                                      |
| ----------- | --------------------------------------------------------------- |
| `--all`     | Получить изменения со всех удалённых репозиториев               |
| `--prune`   | Удалить ссылки на удалённые ветки, которых больше не существует |
| `--tags`    | Получить все теги                                               |
| `--dry-run` | Показать, что будет загружено, без выполнения                   |

`git fetch origin foo` - мы скачиваем [[commit]]ы с [[branch]] `foo` удаленного репозимтория и перемещаем их в нашу [[branch]] `origin/foo`. [[git]] не скачивает изменения в нашу локальную ветвь `foo` что-бы избегать конфликтов, ведь возможно мы сейчас над ней работаем. 

`git fetch origin <источник>:<получатель>` - если  хотите закачать коммиты прямиком в вашу локальную [[branch]], тогда  вы можете явно указать источник и получатель через двоеточние. Вы можете воспользоваться таким приёмом лишь для ветки, на которой вы не находитесь в настоящий момент. 

Если мы не укажем источник и ветку которая не существует `git fetch origin :<получатель>` то [[branch]] будет создана. 

``` bash
git fetch # Получить изменения из origin
git fetch origin # Получить изменения из origin
git fetch --all # Получить изменения со всех remote
git fetch --prune # Удалить устаревшие ссылки на удалённые ветки
git fetch --tags # Загрузить все теги
```

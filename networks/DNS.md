Cистема преобразования доменных имён в IP-адреса. 
DNS использует протокола транспортного уровня - и UDP, и TCP (на порту 53). 

- [[UDP]] используется по умолчанию для большинства стандартных запросов и ответов, так как он обеспечивает максимальную скорость и низкие накладные расходы.
- [[TCP]] применяется, если размер DNS-ответа превышает 512 байт, а также при передаче зон (синхронизации) между серверами. В современных сетях [[TCP]] используется чаще из-за возросшего объема данных (например, для DNSSEC). 

**`Шаг 1. Клиент`**
1. Ты открываешь `google.com`
2. Компьютер проверяет `Кэш DNS` и `/etc/hosts`
3. Если записи нет, запрос уходит DNS-резолверу.  Например `1.1.1.1`
4. Если он тоже не знает, отправляет запрос на сервер стоящий выше по иерархии.S

Типы DNS-записей

| Тип       | Назначение                                      | Пример                            |
| --------- | ----------------------------------------------- | --------------------------------- |
| **A**     | Имя → IPv4                                      | `google.com → 142.250.185.14`     |
| **AAAA**  | Имя → IPv6                                      | `google.com → 2a00:1450:4001:...` |
| **CNAME** | Псевдоним другого имени                         | `www.site.com → site.com`         |
| **MX**    | Почтовый сервер домена                          | `gmail.com → mail.google.com`     |
| **NS**    | DNS-серверы зоны                                | `google.com → ns1.google.com`     |
| **TXT**   | Текстовая информация                            | SPF, DKIM, подтверждение домена   |
| **PTR**   | IP → Имя (обратный DNS)                         | `8.8.8.8 → dns.google`            |
| **SRV**   | Указывает сервис и порт                         | SIP, Kerberos, Kubernetes         |
| **CAA**   | Какие CA могут выпускать сертификаты для домена | Let's Encrypt, DigiCert           |
| **SOA**   | Информация о DNS-зоне                           | Главный NS, серийный номер зоны   |

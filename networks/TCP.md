Tранспортный протокол, обеспечивающий надёжную передачу данных между двумя устройствами. Гарантирует, что данные:
- будут доставлены;
- придут в правильном порядке;
- не будут потеряны незаметно;
- не будут продублированы.

**`Установка TCP-соединения`** происходит в три этапа:
1. `SYN`
	Клиент говорит *Хочу установить соединение*
2. `SYN-ACK`
	Сервер отвечает *Готов принять соединение*
3. `ACK`
	Клиент подтверждает *Отлично, начинаем работу*

**`Передача данных`**
TCP разбивает данные на сегменты.
``` bash
Большой файл
        ↓
TCP Segment 1
TCP Segment 2
TCP Segment 3
...
```
Каждый сегмент получает номер. Благодаря этому получатель может собрать данные обратно в правильном порядке.

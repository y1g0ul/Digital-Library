Cетевой протокол, отвечающий за адресацию устройств и доставку пакетов между сетями.

Основные поля IP-заголовка:
- IP отправителя
- IP получателя
- TTL (Time To Live). Показывает, сколько маршрутизаторов пакет может пройти. Каждый роутер уменьшает TTL на 1.
- Protocol. Указывает, какой протокол находится внутри IP-пакета.
	`6`- [[TCP]]
	`17` - [[UDP]]
	`1` - [[ICMP]]

**`IPv4`**
4 Блока `0-255`, разделенные между собой точками. 
Делятся на:
- Белые (публичные) - доступны из интернета.
- Серые (частные, private) - используются только внутри локальных сетей. Диапазоны частных адресов:
``` bash
10.(0-255).(0-255).(0-255)/8
172.(16-32).(0-255).(0-255)/12
192.168.(0-255).(0-255)/16
```

**`Специальные IPv4-адреса`**
- Broadcast (направленная широковещательная рассылка)
`192.168.1.255/24` - Позволяет отправить один пакет, каоторый будет доставлен всем хостам в адресо ванной сети. 
- Ограниченная широковещательная рассылка (Limited Broadcast)
`255.255.255.255` - Используется для отправки пакета всем устройствам локальной сети, когда отправитель ещё не знает адрес сети. Не сможет пройти через маршрутизатеры. Пакеты получат все хосты и узлы в сети отправителя. 
- Неопределённый адрес (Unspecified Address)
`0.0.0.0` - Означает «У меня ещё нет IP-адреса». Используется в [[DHCP]], как адрес по умолчанию. 
- Хост в текущей сети
`0.0.0.15` - Сегодня практически не используется. 
- Loopback (Петлевой адрес)
`127.0.0.0/8` - Нужен для отправки пакетов на свое же устройство.

**`IPv6`**
32 бита четвертой версии превратили в 128.
- Глобальный префикс маршрутизации ( первые 3 блока / 48 бит) - выдается провайдером или региональным интернет-регистратором (RIR), идентифицирует вашу сеть в глобальном интернете. 
- Идентификатор подсети (4 блок / 16 бит) — используется для разделения сети на более мелкие локальные сегменты внутри вашей организации.
- Идентификатор интерфейса (последние 4 блока / 64 бит) — уникальный адрес конкретного устройства (хоста). Часто формируется автоматически на основе MAC-адреса с помощью технологии EUI-64. 

Виды адресов:
- Unicast - одноадресная передача. При таком типе адресации указывается единственный адрес, и пакет получает только один участник. Unicast-адреса должны быть уникальными в пределах какой-то части сети:
	- Global unicast - уникальны в пределах интернета. 
	- Unique local - уникальны в пределах сайта или организации. 
	- Link-local - уникальны в пределах канала. 
- Anycast - альтернативная передача. При таком типе адресации IP-адрес получателя принадлежит ещё и некоторым другим узлам. Пакет, отправленный на любойcast-адрес, доставляется до одного из участников группы, как правило, до ближайшего к отправителю. 
- Multicast - многоадресная передача на группу. При таком типе адресации указывается, что пакет должен получить не один участник, а группа, и определяется признак, по которому получатель должен обрабатывать содержимое. Например, можно отправить IP-пакет на multicast-группу «все роутеры» или на группу «все получатели первого канала».

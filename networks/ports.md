Числовой идентификатор приложения или сервиса внутри устройства. Нужны для того, что бы система понимала к какому [[процесс]]у предназначены данные. Указываются в служебных данных [[TCP]] и [[UDP]]. Один и тот же номер порта может существовать одновременно и в одном и в другом протоколе. [[networks/IP|IP]] + порт называется [[socket]]. 

**`Диапазоны портов`** Порты это числа 0 - 65535. 
- Well Known Ports (`0 - 1023` Стандартные сервисы)
``` bash
# Пример
22 SSH
53 DNS
80 HTTP
443 HTTPS
```
- Registered Ports (`1024 - 49151` Используются различными приложениями)
``` bash
# Пример
3306 MySQL
5432 PostgreSQL
8080 HTTP-alt
```
- Ephemeral Ports (`49152 - 65535` Временные клиентские порты)
	В сетевом соединении используется порт источника и порт назначения. Когда мы делаем запрос на какой-то сервис, на нашем компьютере открывается случайный порт, который нужен что бы система правильно поймала ответ и отдала его ожидающему процессу приложения. 
``` bash
Источник:
192.168.1.100:54321

Назначение:
142.250.185.14:443
```

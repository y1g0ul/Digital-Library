В [[Linux]] каждому процессу назначается **приоритет выполнения**, который влияет на распределение CPU.

Приоритет управляется через значение **nice**.  
Чем меньше значение nice → тем выше приоритет процесса.

Диапазон nice: **от -20 (высокий приоритет) до 19 (низкий приоритет)**.

**`nice`**
Утилита в [[Linux]] для запуска процесса с изменённым приоритетом (значением nice).
``` bash
nice -n значение команда [аргументы]
```
Если `-n` не указано — используется значение `10`.

|Параметр|Описание|
|---|---|
|`-n N`|Установить значение nice|
|`--help`|Помощь|
|`--version`|Версия|
``` bash
nice -n 10 sleep 100        # запустить с пониженным приоритетом
nice -n 19 tar cf backup.tar /home   # минимальный приоритет
nice -n -5 ./app            # высокий приоритет (только root)
```


**`renice`**
Утилита в [[Linux]] для изменения значения nice уже запущенного процесса.
``` bash
renice значение [-p PID] [-u user] [-g группа]
```

|Параметр|Описание|
|---|---|
|`значение`|Новое значение nice|
|`-p PID`|Изменить для конкретного процесса|
|`-u user`|Изменить для процессов пользователя|
|`-g группа`|Изменить для процессов группы|
``` bash
renice 15 -p 1234          # изменить nice процесса 1234
renice 10 -u user          # изменить для процессов пользователя
renice -5 -p 1234          # повысить приоритет (только root)
ps -o pid,ni,cmd           # проверить значение nice
```

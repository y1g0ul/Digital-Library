 Диспетчер задач в [[Linux]]

## Вкладка main

**`CPU`**
Показывает загрузку каждого ядра процессора.
Цвета могут немного отличаться в зависимости от темы, но обычно:
- `Зелёный` - пользовательские процессы (user)
- `Красный` - системное время ядра (system)
- `Синий` - процессы с низким [[nice]]
- `Оранжевый/Фиолетовый` - ожидание ввода-вывода (I/O wait)
- `Серый` - виртуальные машины (steal) или другие специальные состояния

**`Memory`**
Использование оперативной памяти. Не стоит паниковать, если память занята почти полностью - [[Linux]] активно использует свободную RAM под кэш.

**`Swap`**
Использование раздела подкачки.

**`Tasks`**
- Tasks - количество процессов.
- thr - количество потоков (threads).
- kthr - потоки ядра.

**`Load average`**
Средняя нагрузка за:
- 1 минуту;
- 5 минут;
- 15 минут.
Важно понимать - это не процент загрузки CPU. Он показывает количество процессов, которые:
- выполняются;
- готовы выполняться;
- или ждут ввода-вывода.
Например, на 8-ядерном процессоре:
```
Load = 8
```
означает, что все ядра примерно полностью заняты.

**`Uptime`**
Сколько работает система после последней загрузки.

## Таблица процессов

| Поле    | Что означает                                                                                                                                                                                                             |
| ------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| [[PID]] | Идентификатор процесса.                                                                                                                                                                                                  |
| USER    | Владелец процесса.                                                                                                                                                                                                       |
| PRI     | Фактический приоритет, который ядро Linux использует при планировании задач. Как правило, он вычисляется на основе значения [[nice]]. По умолчанию для обычных процессов значение `PRI` рассчитывается как [[nice]] + 20 |
| NI      | Приоритет, задаваемый [[nice]]                                                                                                                                                                                           |
| VIRT    | Виртуальная память. Общий объем памяти, который [[процесс]] затребовал у системы (включая библиотеки, файлы на диске и неиспользуемый код).                                                                              |
| RES     | Резидентная память. Физическая оперативная память (RAM), которую [[процесс]] занимает прямо сейчас.                                                                                                                      |
| SHR     | Разделяемая память. Часть памяти, которую этот [[процесс]] делит с другими программами (например, общие библиотеки).                                                                                                     |
| S       | Состояние процесса.<br>`R` - Running (выполняется)<br>`S` - Sleeping (ожидает)<br>`D` - Непрерываемое ожидание (обычно I/O)<br>`T` - Остановлен<br>`Z` - Zombie                                                          |
| CPU%    | Процент использования процессора. На многоядерных системах может быть больше 100%.                                                                                                                                       |
| MEM%    | Процент используемой оперативной памяти.                                                                                                                                                                                 |
| TIME+   | Общее процессорное время процесса с момента запуска.<br>Не путать с временем существования процесса.                                                                                                                     |
| Command | Команда запуска процесса.                                                                                                                                                                                                |
## Вкладка I/O

| Поле       | Что означает                                                                                                                                                   |
| ---------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [[PID]]    | Идентификатор [[процесс]]а.                                                                                                                                    |
| USER       | Пользователь, от имени которого запущен [[процесс]].                                                                                                           |
| IO         | Приоритет операций ввода-вывода [[процесс]] (I/O priority). Обычно соответствует настройке [[ionice]]                                                          |
| DISK R/W   | Общий объём данных, прочитанных и записанных процессом с момента его запуска. Это накопительный счётчик, а не текущая скорость.                                |
| DISK READ  | Текущая скорость чтения с диска .                                                                                                                              |
| DISK WRITE | Текущая скорость записи на диск.                                                                                                                               |
| SWAPD%     | Процент времени, который процесс тратит на ожидание операций со swap (подкачкой). Высокие значения могут говорить о нехватке оперативной памяти.               |
| IOD%       | Процент времени, которое процесс находится в ожидании завершения операций ввода-вывода (I/O Delay). Чем выше значение, тем сильнее процесс «упирается» в диск. |
| Command    | Команда, которой был запущен процесс.                                                                                                                          |

Используйте стрелки на клавиатуре для навигации.

Нажмите `q`, чтобы выйти.

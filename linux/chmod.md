Команда в [[Linux]] для управления правами доступа к файлам. Есть несколько способов создания прав:

**Цифровой**
Каждая цифра — сумма прав:  
4 = `r` (чтение)  
2 = `w` (запись)  
1 = `x` (выполнение)  

Примеры:
`chmod 755 file.txt` →  
- Владелец: `7` (4+2+1 = `rwx`)  
- Группа: `5` (4+1 = `r-x`)  
- Остальные: `5` (4+1 = `r-x`)  
`chmod 644 file.txt` →  
- Владелец: `rw-`  
- Группа и остальные: `r--`  

**Буквенный**
`u` (user), `g` (group), `o` (others), `a` (all)  
`+` (добавить), `-` (убрать), ` = `(установить точно) 
Примеры:
- `chmod u+x script.sh` — дать владельцу право на выполнение.  
- `chmod go-w file.txt` — запретить запись группе и остальным.  
- `chmod a=rw file.txt` — всем: чтение + запись (но не выполнение).  
 - `chmod u+x,g+x file` — дать владельцу и группе право на выполнение.
 
Важные примеры:
- `chmod 700 file` — только владелец может читать/писать/выполнять.  
- `chmod +x script.sh` — сделать файл выполняемым для всех.  
- `chmod -R 755 folder/` — рекурсивно (для всех файлов внутри папки).  
- `chmod -w file` — убирает права на запись у владельца и группы


Если мы уберем у директории права на чтение то мы не сможем узнать что внутри нее, но если мы знаем имя файла внутри нее мы все равно сможем к нему обратиться.

``` bash
y1g0ul@DESKTOP-195S4U1:~$ chmod a-r dir/
y1g0ul@DESKTOP-195S4U1:~$ ls -l dir/
ls: cannot open directory 'dir/': Permission denied
y1g0ul@DESKTOP-195S4U1:~$ ls -l dir/file
-rw-r--r-- 1 y1g0ul y1g0ul 0 Dec 17 02:45 dir/file
```

Права на использование директории позволяют считывать или не считывать информацию из [[inode]] файла.

``` bash
y1g0ul@DESKTOP-195S4U1:~$ chmod a-x dir/
y1g0ul@DESKTOP-195S4U1:~$ ls -l dir/
ls: cannot access 'dir/file': Permission denied
total 0
-????????? ? ? ? ?            ? file
```


Вопрос: В домашней директории пользователя администратор root создает файл. Соответственно у этого файла будет группа root и владлец root. После чего убирает у него вообще все права. 

Сможет ли пользователь удалить такой файл?

Ответ - `Да`. Удаление это работа с каталогом в котором находится директория или файл. Удаление определяется правом на каталог. Перезапись файла и обновление какой-то информации это работа уже с самим файлом.

Планировщик задач в [[Linux]] который используется как дополнение к класическому [[cron]].  В отличие от него, представляющего обычный список, хранит информацию и может проверять пропускал ли он задачи. Предназначен для компьютеров и серверов, которые не работают круглосуточно и гарантирует выполнение.
``` bash
anacron [опции]
```

|Ключ|Что делает|
|---|---|
|`-f`|запустить задачи принудительно|
|`-n`|запустить задачи сразу, без задержек|
|`-d`|отладочный режим (не уходить в фон)|
|`-s`|выполнять задачи последовательно|
|`-u`|обновить временные метки задач|
|`-T`|проверить конфигурационный файл|
``` bash
anacron                       # запустить anacron

anacron -f                    # принудительно выполнить задачи

anacron -n                    # выполнить сразу

anacron -d                    # отладочный режим

anacron -fn                   # принудительно и без задержек

anacron -T                    # проверить конфигурацию
```

Конфигурация находится  в `/etc/anacrontab`. 
``` bash
# /etc/anacrontab

SHELL=/bin/sh # Какая оболочка будет запускать команды.
PATH=/sbin:/bin:/usr/sbin:/usr/bin # Пути для поиска программ.
MAILTO=root # Кому отправлять вывод задач по почте.

# период  задержка  идентификатор  команда
1       5       cron.daily      run-parts /etc/cron.daily
7       10      cron.weekly     run-parts /etc/cron.weekly
@monthly 15     cron.monthly    run-parts /etc/cron.monthly
```
- **`Период`** - через сколько дней запускать задачу
- **`Задержка`** - сколько минут подождать после запуска anacron
- **`Идентификатор`**  - уникальное имя задания. `Anacron` хранит информацию о последнем запуске в `/var/spool/anacron/`.
- **`Команда`** - что запускать. Обчно используется с [[run-parts]].

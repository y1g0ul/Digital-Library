Команда для просмотра файлов в [[Linux]]

> [!note] Я часто использую опции 
> `-l` (long listing format - вывод в колонку с дополнительной информацией о файлах), `-t` (сортировка по времени изменения файла/директории) и `-r` (обратная сортировка - в сочетании с `-t` наиболее **«свежие»** файлы будут внизу. И еще можно использовать опцию `-h` - вывод в **human readable** формате.

Флаг `-a` использует для просмотра **скрытых** файлов. Скрытым файлом в [[Linux]] считается тот имя которого начинается с точки.
Флаг `-d` показывает сами каталоги, а не их содержимое.
Флаг `-i` показывает номера [[inode]]

Если атрибут файла начинается с :
- **`d`** это директория/каталог.
- **`-`** это обычный файл
- **`l`** это ссылка
- **`b`** это блочное устройство
- **`c`** это символьное устройство
- **`s`** это сокет

Существует несколько разновидностей утилит родственников:

**`lsmod`**
- загруженные **модули ядра**
- по сути: `ls` для /[[proc]]/modules

	`modinfo`
	- показывает **информацию о конкретном модуле**
	- версия, автор, описание, параметры
	
	`insmod`
	- загружает модуль вручную
	- без проверки зависимостей
	
	`rmmod`
	- выгружает модуль
	- если модуль используется → ошибка
	
	`modprobe` 
	- **корректно** загружает и выгружает модули
	- автоматически учитывает зависимости

**`lspci`**
- устройства на шине **PCI / PCIe**
- видеокарта, сеть, звук

**`lsusb`**
- **USB**-устройства
- флешки, мыши, камеры

**`lsblk`**
- диски и разделы
- показывает структуру хранения данных

**`lsipc`**
- IPC-объекты
- shared memory, семафоры, очереди сообщений

**`lscpu`**
- информация о **процессоре**
- ядра, потоки, архитектура

**`lsmem`**
- информация о **памяти**
- NUMA, области RAM

**`lslocks`**
- активные блокировки файлов

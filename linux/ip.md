# ip

Команда в [[Linux]] для управления сетевыми интерфейсами, IP-адресами, маршрутами, ARP/NDP-кэшем и другими параметрами сети. Входит в пакет **iproute2** и является современной заменой команд `ifconfig`, `route` и `arp`.

```bash
ip [объект] [команда] [параметры]
```

|Объект|Что делает|
|---|---|
|`addr` (`a`)|Управление IP-адресами|
|`link` (`l`)|Управление сетевыми интерфейсами|
|`route` (`r`)|Просмотр и изменение маршрутов|
|`neigh` (`n`)|Просмотр и управление ARP/NDP-кэшем|
|`rule`|Управление правилами маршрутизации|
|`monitor`|Мониторинг изменений сетевой конфигурации|
|`netns`|Управление сетевыми пространствами имён|
|`tunnel`|Управление сетевыми туннелями|

**`Работа с IP-адресами`**
```bash
ip addr show # Показать все IP-адреса
ip addr show eth0 # Показать адреса интерфейса eth0
ip -br addr # Краткий вывод IP-адресов
sudo ip addr add 192.168.1.100/24 dev eth0 # Добавить IP-адрес
sudo ip addr del 192.168.1.100/24 dev eth0 # Удалить IP-адрес
sudo ip addr flush dev eth0 # Удалить все IP-адреса интерфейса
```

**`Работа с сетевыми интерфейсами`**
```bash
ip link show # Показать интерфейсы
ip -br link # Краткий список интерфейсов
sudo ip link set eth0 up # Включить интерфейс
sudo ip link set eth0 down # Выключить интерфейс
sudo ip link set eth0 mtu 9000 # Изменить MTU
sudo ip link set eth0 name lan0 # Переименовать интерфейс
```

**`Работа с маршрутами`**
```bash
ip route # Показать таблицу маршрутизации
ip route get 8.8.8.8 # Показать маршрут до узла
sudo ip route add 192.168.2.0/24 via 192.168.1.1 # Добавить маршрут
sudo ip route add default via 192.168.1.1 # Добавить маршрут по умолчанию
sudo ip route del 192.168.2.0/24 # Удалить маршрут
```

**`Работа с ARP/NDP-кэшем`**
```bash
ip neigh # Показать ARP/NDP-кэш
ip neigh show dev eth0 # Показать кэш интерфейса
sudo ip neigh del 192.168.1.10 dev eth0 # Удалить запись
sudo ip neigh flush dev eth0 # Очистить кэш
```

**`Мониторинг`**
```bash
ip monitor # Следить за всеми изменениями
ip monitor link # Следить за интерфейсами
ip monitor address # Следить за изменением IP-адресов
ip monitor route # Следить за маршрутами
```

**`Краткие формы`**

|Полная команда|Сокращение|
|---|---|
|`ip addr`|`ip a`|
|`ip link`|`ip l`|
|`ip route`|`ip r`|
|`ip neigh`|`ip n`|

> [!info] Важно
> Команда `ip` изменяет настройки **только до следующей перезагрузки**. Чтобы сохранить их постоянно, используют:
>
> - Netplan
> - NetworkManager
> - systemd-networkd

## Полезно знать

> [!tip]
> В повседневной работе системного администратора чаще всего используются четыре подкоманды:
>
> - `ip addr` — работа с IP-адресами;
> - `ip link` — управление интерфейсами;
> - `ip route` — просмотр и настройка маршрутов;
> - `ip neigh` — просмотр ARP/NDP-кэша.
>
> В большинстве случаев именно они полностью заменяют устаревшие команды `ifconfig`, `route` и `arp`.

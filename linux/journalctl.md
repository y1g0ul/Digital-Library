**`systemd-journald`** — это единый сервис, который собирает все логи системы.
- **база логов**, а не просто текстовый файл
- хранится в бинарном формате
- можно фильтровать по:
    - сервису ([[unit]])
    - времени
    - уровню ошибок
    - [[PID]]
    - [[UID]]
    - [[unit]]

Папки хранения:
- `/run/log/journal/` - временно (пропадают после reboot)
- `/var/log/journal/` - постоянно (если директория существует)
- `/etc/systemd/journald.conf` - настройки

| Параметр в настройках   | Назначение                                                                         |
| ----------------------- | ---------------------------------------------------------------------------------- |
| `Storage=`              | Где хранить журналы: `volatile` (RAM), `persistent` (диск), `auto` (автоматически) |
| `Compress=`             | Сжимать старые журналы для экономии места                                          |
| `SyncIntervalSec=`      | Как часто принудительно сбрасывать журналы на диск                                 |
| `RateLimitIntervalSec=` | Интервал для ограничения потока сообщений                                          |
| `RateLimitBurst=`       | Максимум сообщений за интервал до начала отбрасывания                              |
| `SystemMaxUse=`         | Максимальный объём журналов на диске                                               |
| `SystemKeepFree=`       | Сколько свободного места всегда оставлять на диске                                 |
| `SystemMaxFileSize=`    | Максимальный размер одного файла журнала                                           |
| `SystemMaxFiles=`       | Максимальное количество файлов журналов                                            |
| `MaxRetentionSec=`      | Максимальный срок хранения журналов                                                |
| `MaxFileSec=`           | Через какое время создавать новый файл журнала                                     |
| `ForwardToSyslog=`      | Пересылать сообщения в rsyslog/syslog-ng                                           |
| `ForwardToConsole=`     | Выводить сообщения на системную консоль                                            |
| `MaxLevelStore=`        | Максимальный уровень сообщений, сохраняемых в журнал                               |
| `ReadKMsg=`             | Читать сообщения ядра (`/proc/kmsg`)                                               |

```
journalctl [OPTIONS] [-p PRIORITY] [-u UNIT] [--since TIME] [--until TIME] [_MATCHES...]
```
- **`[OPTIONS]`** — ключи управления выводом
- **`[-p PRIORITY]`** — фильтр по уровню важности
- **`[-u UNIT]`** — фильтр по сервису / юниту systemd
- **`[--since / --until TIME]`** — фильтр по времени
- **`[_MATCHES...]`** — фильтры по PID, UID, имени процесса или бинарнику

| Ключ            | Что делает                              |
| --------------- | --------------------------------------- |
| `-b`            | журналы текущей загрузки                |
| `-b -1`         | Логи предыдущей загрузки                |
| `-k`            | только сообщения ядра                   |
| `-u`            | журналы сервиса                         |
| `-f`            | следить за журналом в реальном времени  |
| `-n`            | показать последние N записей            |
| `-r`            | вывод в обратном порядке                |
| `-p`            | фильтр по приоритету                    |
| `-o short`      | Краткий формат                          |
| `-o verbose`    | Полный формат со всеми полями           |
| `-xe`           | подробные ошибки и конец журнала        |
| `--since`       | записи начиная с указанного времени     |
| `--until`       | записи до указанного времени            |
| `--disk-usage`  | размер журналов                         |
| `--vacuum-size` | удалить журналы до указанного размера   |
| `--vacuum-time` | удалить журналы старше указанного срока |
| `--no-pager`    | Вывод сразу в терминал, без `less`      |
| `--list-boots`  | Список всех загрузок                    |

Уровни важности (PRIORITY)

| Ключ    | Значение           |
| ------- | ------------------ |
| emerg   | Критическая авария |
| alert   | Срочная проблема   |
| crit    | Критическая ошибка |
| err     | Ошибка             |
| warning | Предупреждение     |
| notice  | Важная информация  |
| info    | Информация         |
| debug   | Отладка            |

Фильтры по процессу / метаданные (MATCHES)

| Фильтр                      | Что делает         |
| --------------------------- | ------------------ |
| `_SYSTEMD_UNIT=ssh.service` | По юниту / сервису |
| `_PID=1234`                 | По процессу        |
| `_UID=1000`                 | По пользователю    |
| `_COMM=sshd`                | По имени команды   |
| `_EXE=/usr/sbin/sshd`       | По бинарнику       |
``` bash
journalctl

journalctl -b
# текущая загрузка

journalctl -b -1
# предыдущая загрузка

journalctl -k
# сообщения ядра

journalctl -u ssh
# журналы ssh

journalctl -u nginx

journalctl -f
# аналог tail -f

journalctl -n 50
# последние 50 записей

journalctl -p err
# только ошибки

journalctl --since today

journalctl --since "1 hour ago"

journalctl --since yesterday

journalctl --since "2026-06-15 10:00"

journalctl --since today --until now
```

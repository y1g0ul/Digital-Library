Команда в [[Linux]] для диагностики сетевого соединения. Объединяет возможности [[ping]] и [[traceroute]], показывая маршрут до узла и статистику потерь пакетов и задержек в режиме реального времени.

```bash
mtr [опции] хост
```

|Ключ|Что делает|
|---|---|
|`-r`|Вывести отчет и завершить работу|
|`-c`|Количество отправляемых запросов|
|`-w`|Широкий формат вывода|
|`-n`|Не выполнять DNS-разрешение|
|`-4`|Использовать только IPv4|
|`-6`|Использовать только IPv6|
|`-i`|Интервал между запросами|
|`-u`|Использовать UDP-пакеты|
|`-T`|Использовать TCP SYN-пакеты|
|`-P`|Порт назначения для TCP/UDP|

**`Основные примеры`**

```bash
mtr google.com # Интерактивная диагностика
mtr 8.8.8.8 # Проверка по IP
mtr -n 8.8.8.8 # Без DNS-разрешения
mtr -4 google.com # Использовать IPv4
mtr -6 google.com # Использовать IPv6
```

**`Создание отчета`**

```bash
mtr -r google.com # Отчет и завершение
mtr -r -c 20 google.com # Отчет после 20 запросов
mtr -rw google.com # Широкий отчет
```

**`Использование TCP и UDP`**

```bash
mtr -T google.com # Использовать TCP SYN
mtr -u google.com # Использовать UDP
mtr -T -P 443 google.com # TCP на порт 443
```

**`Пример вывода`**

```text
Host               Loss%  Snt  Last  Avg  Best  Wrst StDev
1. 192.168.1.1      0.0%   10   1.2  1.3   1.0   2.0   0.3
2. 10.0.0.1         0.0%   10  10.5 10.4  10.2  11.0   0.2
3. 8.8.8.8          0.0%   10  23.4 23.7  22.9  25.1   0.5
```

**`Поля вывода`**

|Поле|Описание|
|---|---|
|`Loss%`|Процент потерянных пакетов|
|`Snt`|Количество отправленных запросов|
|`Last`|Время последнего ответа|
|`Avg`|Средняя задержка|
|`Best`|Минимальная задержка|
|`Wrst`|Максимальная задержка|
|`StDev`|Разброс времени ответа|

> [!info] Как работает mtr
> `mtr` непрерывно отправляет пакеты к каждому узлу маршрута и одновременно собирает статистику задержек и потерь. В отличие от [[traceroute]], результаты постоянно обновляются, что позволяет обнаружить нестабильные соединения.

> [!tip] Полезно знать
> Для сохранения результатов диагностики обычно используют:
>
> - `mtr -r` — текстовый отчет;
> - `mtr -r -c 100` — более точная статистика после 100 запросов.
>
> Если промежуточный маршрутизатор показывает высокий `Loss%`, а следующие узлы отвечают без потерь, это **не обязательно означает проблему**. Многие маршрутизаторы ограничивают или игнорируют ответы на диагностические пакеты, продолжая при этом нормально пересылать обычный трафик.

Служба в [[Linux]] для настройки и управления сетевыми интерфейсами. Чаще всего используется на серверах и применяется [[netplan]] в качестве backend.

``` bash
networkctl <команда>
```

| Команда | Что делает |
|---------|------------|
| `list` | показать сетевые интерфейсы |
| `status` | подробная информация об интерфейсах |
| `status <интерфейс>` | информация о конкретном интерфейсе |
| `up <интерфейс>` | поднять интерфейс |
| `down <интерфейс>` | отключить интерфейс |
| `reload` | перечитать конфигурацию |
| `renew <интерфейс>` | обновить DHCP-аренду |
| `reconfigure <интерфейс>` | заново применить конфигурацию |

Конфигурационные файлы:

```text
/etc/systemd/network/
```

Основные службы:

| Служба | Назначение |
|--------|------------|
| `systemd-networkd` | настраивает сетевые интерфейсы |
| `systemd-resolved` | управляет DNS |
| `networkctl` | управление и просмотр состояния сети |

Примеры:

``` bash
systemctl status systemd-networkd     # проверить работу службы
systemctl restart systemd-networkd    # перезапустить службу

networkctl list                       # список интерфейсов
networkctl status                     # состояние сети
networkctl status enp0s3              # состояние интерфейса
networkctl reload                     # перечитать конфигурацию
networkctl reconfigure enp0s3         # применить конфигурацию заново
```

> **systemd-networkd** обычно используется на серверах, где не требуется управление Wi-Fi или графический интерфейс. В Ubuntu Server именно он чаще всего применяется [[netplan]] для настройки сети.

Команда в [[Linux]] для работы с TCP/UDP соединениями (netcat)
``` bash
nc [опции] хост порт
```

|Ключ|Что делает|
|---|---|
|`-l`|режим прослушивания (server mode)|
|`-p`|указать локальный порт|
|`-v`|подробный вывод|
|`-z`|сканирование порта без отправки данных|
|`-u`|использовать UDP|
|`-n`|не использовать DNS|
|`-k`|не закрывать сокет после подключения|
|`-w`|таймаут соединения|
|`-e`|выполнить программу после подключения (не везде доступно)|
``` bash

nc google.com 80                  # подключиться к 80 порту
nc -zv 127.0.0.1 22               # проверить открыт ли порт
nc -zv 127.0.0.1 1-1000           # сканирование диапазона портов
nc -l 4444                        # слушать порт 4444
nc -lv 4444                       # слушать с подробным выводом
echo "hello" | nc 127.0.0.1 4444  # отправить данные
nc -u 127.0.0.1 53                # UDP соединение
nc -w 5 google.com 80             # таймаут 5 секунд
```

Можно отправить файл, перенаправив вывод команды. На принимающей стороне сохраняют поток: `nc -l 1234 > file.txt`, а на отправляющей: `nc 192.168.1.10 1234 < file.txt`.

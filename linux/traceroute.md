Команда в [[Linux]] для определения маршрута прохождения пакетов до удалённого узла. Показывает все промежуточные маршрутизаторы (hop'ы) и время прохождения до каждого из них.

```bash
traceroute [опции] хост
```

|Ключ|Что делает|
|---|---|
|`-I`|использовать ICMP вместо UDP|
|`-T`|использовать TCP SYN-пакеты|
|`-m`|максимальное количество переходов (TTL)|
|`-w`|время ожидания ответа|
|`-q`|количество запросов на каждый hop|
|`-p`|порт назначения (UDP/TCP)|
|`-n`|не выполнять DNS-разрешение адресов|
|`-4`|использовать только IPv4|
|`-6`|использовать только IPv6|

**`Основные примеры`**

```bash
traceroute google.com # Построить маршрут до узла
traceroute 8.8.8.8 # Построить маршрут по IP
traceroute -n 8.8.8.8 # Не выполнять DNS-разрешение
traceroute -I google.com # Использовать ICMP
traceroute -T google.com # Использовать TCP SYN
```

**`Ограничение маршрута`**

```bash
traceroute -m 10 google.com # Максимум 10 переходов
traceroute -q 1 google.com # Один запрос на каждый hop
traceroute -w 2 google.com # Ожидать ответ 2 секунды
```

**`Работа с IPv4 и IPv6`**

```bash
traceroute -4 google.com # Использовать IPv4
traceroute -6 google.com # Использовать IPv6
```

**`Пример вывода`**

```text
traceroute to google.com (142.250.74.14), 30 hops max

1  192.168.1.1       1.2 ms   1.1 ms   1.0 ms
2  10.0.0.1         10.5 ms  10.4 ms  10.6 ms
3  172.16.1.1       18.3 ms  18.5 ms  18.4 ms
4  142.250.74.14    25.8 ms  25.6 ms  25.7 ms
```

**`Обозначения вывода`**

|Поле|Описание|
|---|---|
|Hop|Номер перехода (маршрутизатора)|
|IP/Hostname|Адрес или имя маршрутизатора|
|`ms`|Время прохождения пакета|
|`*`|Ответ не получен (таймаут или фильтрация)|

> [!info] Как работает traceroute
> Команда отправляет пакеты с постепенно увеличивающимся значением **TTL (Time To Live)**.
>
> Каждый маршрутизатор уменьшает TTL на 1. Когда TTL становится равен 0, маршрутизатор возвращает сообщение **ICMP Time Exceeded**. Благодаря этому `traceroute` определяет весь путь следования пакета до конечного узла.

> [!tip] Полезно знать
> По умолчанию в Linux `traceroute` использует **UDP-пакеты**. Если они блокируются брандмауэром, можно использовать:
>
> - `-I` — ICMP (аналогично `ping`);
> - `-T` — TCP SYN, который часто проходит через межсетевые экраны.

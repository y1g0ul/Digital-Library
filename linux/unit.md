Это объект [[Linux]], которым управляет [[systemd]].

**Службы (демоны)** **`service`**
Это программы, которые работают в фоне.
Примеры:
- `sshd.service` — SSH
- `nginx.service` — веб-сервер
- `docker.service` — Docker

**Точки монтирования** **`mount`**
[[systemd]] может управлять монтированием файловых систем.
Пример:
- `/home` → `home.mount`
- `/mnt/data` → `mnt-data.mount`

**Устройства** **`device`**
Юниты, которые появляются, когда ядро обнаруживает устройство.
Примеры:
- диск
- USB-флешка
- сетевой интерфейс

**Сокеты (ленивый запуск сервисов)** **`socket`**
Сервис **не запускается сразу**, пока не появился запрос.
Пример:
- `sshd.socket` слушает порт 22
- когда кто-то подключается → запускается `sshd.service`

[[unit]]-файлы лежат в:
`/usr/lib/systemd/`
- [[unit]] по умолчанию, которые ставятся вместе с пакетами.
- трогать **не рекомендуется**
- при обновлении могут перезаписаться

`/etc/systemd/`
Мои юниты и мои изменения
- переопределения
- кастомные сервисы
- конфиги администратора
Всегда имеет **приоритет** над `/usr/lib/systemd`.

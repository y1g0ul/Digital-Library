Каталог в [[Linux]] это специальный файл который хранит в себе имена других файлов или каталогов. 

Как это работает:
- Каталог (папка) — это просто список записей вида:  
   `"имя файла" → номер inode`.  
   
- Inode — это «паспорт» файла, где хранится вся информация о нём (права, размер, владелец, где на диске лежат данные и т.д.), кроме имени.  Понятнее будет на примере создания файла: 
	1. Нам нужно сохранить даные на диске физически
	2. Сообщить файловой системе в каком месте эти данные лежат, что-бы файловая система не дала кому нибудь другому это место для записи в будующем
	3. Нам ужно это как-то назвать (файл - это именованная область данных на диске)
	Данные из файла хранятся на диске, а мета информация о данных хранятся в определенном блоке. Этот блок и есть **inode**

- Имя файла нужно только нам, людям, чтобы обращаться к файлу. Для системы важны inode.  

`-rw-rw-r-- 1 myuser myuser 5 Apr  1 09:35 test`

В каталоге хранится только информация о названии файла - `test` и номер **inode**
Вся остальная информация  - `-rw-rw-r-- 1 myuser myuser 5 Apr 1 09:35` подтягивается из **inode**

**inode** имеют свои уникальные имена в пределах одного диска.

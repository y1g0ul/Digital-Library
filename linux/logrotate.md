Команда в [[Linux]] для автоматической ротации, сжатия и удаления логов.  Используется в ежедневном планировщие [[cron]]. 
``` bash
logrotate [опции] конфигурация
```

| Ключ      | Что делает                            |
| --------- | ------------------------------------- |
| `-d`      | отладочный режим (ничего не изменяет) |
| `-f`      | принудительно выполнить ротацию       |
| `-v`      | подробный вывод                       |
| `-s`      | указать файл состояния                |
| `--state` | то же самое, что `-s`                 |
Основной конфигурационный файл `/etc/logrotate.conf`. Дополнительные конфигурации `/etc/logrotate.d/`.  Состояние ротации хранится в `/var/lib/logrotate/status`. По этому файлу logrotate понимает,  когда лог ротировался последний раз. 

|Директива|Что делает|
|---|---|
|`daily`|ротация каждый день|
|`weekly`|ротация каждую неделю|
|`monthly`|ротация каждый месяц|
|`yearly`|ротация каждый год|
|`rotate N`|хранить N старых логов|
|`compress`|сжимать старые логи|
|`delaycompress`|сжимать начиная со следующей ротации|
|`create`|создать новый лог после ротации|
|`copytruncate`|обрезать лог без остановки процесса|
|`missingok`|не выдавать ошибку, если лог отсутствует|
|`notifempty`|не ротировать пустые файлы|
|`size`|ротировать при достижении размера|
|`maxsize`|максимальный размер файла|
|`postrotate`|выполнить команду после ротации|
|`prerotate`|выполнить команду перед ротацией|
Примеры конфигурации:
``` bash
/var/log/app.log {
    daily
    rotate 7
    compress
}
```

``` bash
/var/log/app.log {
    daily
    rotate 7
    compress
}
```

``` bash
/var/log/nginx/*.log {
    weekly
    rotate 4

    postrotate
        systemctl reload nginx
    endscript
}
```

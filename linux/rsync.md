Команда в [[Linux]] для синхронизации и копирования файлов и каталогов локально или по сети. В отличие от [[cp]], rsync копирует только изменённые части файлов, что значительно экономит время и сетевой трафик.

```
Источник
    ↓
Сравнение файлов
    ↓
Передача только изменений
    ↓
Назначение
```

``` bash
rsync [опции] источник назначение
```

|Ключ|Что делает|
|---|---|
|`-a`|режим архива (сохраняет права, владельца, время и рекурсивно копирует)|
|`-v`|подробный вывод|
|`-h`|человекочитаемый размер файлов|
|`-z`|сжимать данные при передаче|
|`-P`|показывать прогресс и продолжать прерванную передачу|
|`-r`|рекурсивное копирование|
|`-u`|не перезаписывать более новые файлы|
|`-n`|пробный запуск без изменений (dry-run)|
|`--delete`|удалить в назначении файлы, отсутствующие в источнике|
|`--exclude`|исключить файлы или каталоги|
|`--progress`|показывать прогресс передачи|
|`-e`|использовать указанную удалённую оболочку (например, SSH)|


``` bash
rsync file.txt backup/

rsync -av source/ backup/

rsync -avh source/ backup/

rsync -avP source/ backup/

rsync -av --delete source/ backup/

rsync -av --exclude "*.log" source/ backup/

rsync -av -e ssh source/ user@server:/backup/

rsync -av user@server:/backup/ local/

rsync -avn source/ backup/    # пробный запуск
```

  Особенность работы с каталогами:
  - Без завершающего /:
``` bash
     rsync -av dir backup/
     # Результат:
     backup/
      └── dir/
```
- С завершающим /:
``` bash 
rsync -av dir/ backup/
# Результат: 
backup/
├── file1
├── file2
└── ...
```

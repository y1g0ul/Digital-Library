Служба в [[Linux]] для сбора, фильтрации, хранения и пересылки системных журналов
``` bash
rsyslogd
```

|Параметр|Что делает|
|---|---|
|`-d`|отладочный режим|
|`-n`|не переходить в фоновый режим|
|`-f`|использовать указанный конфиг|
|`-i`|указать PID-файл|
|`-v`|показать версию|
|`-N1`|проверить конфигурацию|
Основной конфигурационный файл `/etc/rsyslog.conf`.  Дополнительные конфигурации `/etc/rsyslog.d/`.

**`Facilities`**
Источник сообщения.
`auth` - аутентификация
`authpriv` - приватная аутентификация
`cron` - cron и anacron
`daemon` - системные демоны
`kern` - ядро Linux
`mail` - почтовые сервисы
`syslog` - сам rsyslog
`user` - пользовательские программы
`local0-local7` - пользовательские категории

**`Priorities`**
Уровни важности сообщений. Есть в [[journalctl]]

``` 
Приложение
      ↓
  syslog API
      ↓
   rsyslog
      ↓
Файл / Сеть / БД
```

rsyslog получает сообщения от:
- ядра Linux
- системных служб
- приложений
- других серверов по сети
и сохраняет их в файлы или пересылает дальше.

Формат записи:
``` bash
селектор     действие
*.info        /var/log/messages # Все сообщения уровня info и выше записывать в /var/log/messages
*.*    /var/log/all.log # Все сообщения
*.err  /var/log/errors.log # Только ошибки
authpriv.*  /var/log/auth.log # Журнал аутентификации
cron.*      /var/log/cron.log # Сообщения cron
```

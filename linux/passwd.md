Файд /etc/passwd содержит

`user1:x:1001:1001:UserOne:/home/user1:/bin/bash`

Каждая строка соответствует одному **пользователю** и состоит из семи **полей**, разделённых двоеточиями (`:`):  
1. Имя пользователя (логин)  
2. Пароль (раньше хранился здесь, теперь обычно `x`, а реальный хэш — в `/etc/shadow`)  
3. [[uid]] (User ID)  
4. [[gid]] (Group ID)  
5. Комментарий (обычно ФИО или описание)  
6. Домашний каталог
7. Оболочка (shell, например, `/bin/bash`)  


Так же в [[Linux]] существует одноименная команда для изменения пароля пользователя и управления сроком его действия
``` bash
passwd [опции] [пользователь]
```

|Ключ|Что делает|
|---|---|
|`-d`|удалить пароль пользователя|
|`-e`|принудительно потребовать смену пароля при следующем входе|
|`-l`|заблокировать пароль пользователя|
|`-u`|разблокировать пароль пользователя|
|`-S`|показать статус пароля|
|`-n`|минимальное количество дней между сменами пароля|
|`-x`|максимальное количество дней действия пароля|
|`-w`|за сколько дней предупреждать об истечении пароля|
|`-i`|заблокировать аккаунт через N дней после истечения пароля|
``` bash
passwd                         # изменить свой пароль

passwd y1g0ul                  # изменить пароль пользователя

passwd -l y1g0ul               # заблокировать пароль (не удаляет пароль, а делает вход по нему невозможным)

passwd -u y1g0ul               # разблокировать пароль

passwd -d y1g0ul               # удалить пароль

passwd -e y1g0ul               # потребовать смену пароля

passwd -S y1g0ul               # посмотреть статус пароля

passwd -x 90 y1g0ul            # пароль действует 90 дней

passwd -n 7 y1g0ul             # минимум 7 дней между сменами

passwd -w 14 y1g0ul            # предупреждать за 14 дней
```

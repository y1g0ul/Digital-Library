Команда в [[Linux]] для выполнения DNS-запросов. Используется для получения информации о DNS-записях домена, проверки работы DNS-серверов и диагностики проблем с разрешением имён.

```bash
dig [@DNS-сервер] [домен] [тип_записи]
```

|Ключ|Что делает|
|---|---|
|`@`|Использовать указанный DNS-сервер|
|`+short`|Вывести только ответ без служебной информации|
|`+trace`|Показать полный путь поиска DNS-записи|
|`+noall +answer`|Показать только секцию ответа|
|`-x`|Обратный DNS-запрос (Reverse DNS)|
|`-4`|Использовать только IPv4|
|`-6`|Использовать только IPv6|

**`Основные примеры`**

```bash
dig google.com # Получить A-запись
dig google.com AAAA # Получить IPv6-адрес
dig google.com MX # Получить MX-записи
dig google.com NS # Получить DNS-серверы
dig google.com TXT # Получить TXT-записи
dig google.com CNAME # Получить CNAME-запись
```

**`Использование DNS-сервера`**

```bash
dig @8.8.8.8 google.com # Использовать Google DNS
dig @1.1.1.1 google.com # Использовать Cloudflare DNS
dig @192.168.1.1 google.com # Использовать локальный DNS
```

**`Краткий вывод`**

```bash
dig +short google.com # Только IP-адрес
dig +short google.com MX # Только MX-записи
dig +short google.com NS # Только NS-записи
```

**`Обратный DNS-запрос`**

```bash
dig -x 8.8.8.8 # Определить доменное имя по IP
```

**`Трассировка DNS`**

```bash
dig +trace google.com # Показать полный путь поиска записи
```

**`Пример вывода`**

```text
;; QUESTION SECTION:
;google.com.        IN      A

;; ANSWER SECTION:
google.com.  300 IN A 142.250.74.14
```

**`Основные типы DNS-записей`**

|Тип|Описание|
|---|---|
|`A`|IPv4-адрес|
|`AAAA`|IPv6-адрес|
|`MX`|Почтовый сервер|
|`NS`|DNS-серверы домена|
|`CNAME`|Псевдоним домена|
|`TXT`|Текстовая запись|
|`SOA`|Основная информация о DNS-зоне|
|`PTR`|Обратное разрешение IP → домен|

> [!info] Как работает dig
> `dig` отправляет DNS-запрос выбранному DNS-серверу и отображает полученный ответ. По умолчанию используется DNS-сервер, настроенный в системе (например, из `/etc/resolv.conf`).

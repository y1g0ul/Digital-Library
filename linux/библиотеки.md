**`Динамическая линковка`**
Программа не содержит код библиотек, но знает их имена. При запуске они подгружаются из системы.

**`Cтатическая линковка`**
Код библиотек вшит прямо в бинарник. Программа больше по размеру, но работает везде, даже без библиотек.

Динамический линковщик - обычно `/lib64/ld-linux-x86-64.so.2`
Когда пишем [[ls]] происходит так:
1. Ядро видит это **динамически связанный** бинарник
2. Ядро запускает `ld-linux-x86-64.so.2`
3. Линковщик:
    - читает, какие `.so` нужны
    - ищет их в:
		- `/lib`
		- `/usr/lib`
		- `/etc/ld.so.cache`
	- подгружает их в память
4. И только потом запускается [[ls]]

| Каталог                  | Назначение                                                 |
| ------------------------ | ---------------------------------------------------------- |
| `/lib`, `/lib64`         | базовые библиотеки в [[Linux]](нужны при загрузке системы) |
| `/usr/lib`, `/usr/lib64` | пользовательские библиотеки                                |
| `/usr/local/lib`         | вручную установленные                                      |
| `/lib/x86_64-linux-gnu`  | distro-специфично (Debian/Ubuntu)                          |
`ldconfig` - обновить кеш
- читает конфиги
```
/etc/ld.so.conf
/etc/ld.so.conf.d/*.conf
```
- сканирует указанные директории
- создаёт кэш `/etc/ld.so.cache`

`ldd /bin/<bin_name>` - определяет какими библиотеками пользуется программа

Команда для сборки [[image]] в [[docker]].
``` shell
docker  build [OPTIONS] PATH | URL | -
```
- `-t <имя:тег>` - дать имя образу

Основные инструкции

| Инструкция             | Что делает                                                     |
| ---------------------- | -------------------------------------------------------------- |
| `FROM <image>`         | Базовый образ, от которого строим                              |
| `COPY <src> <dest>`    | Копирует файлы из контекста внутрь образа                      |
| `RUN <команда>`        | Выполняет команду при сборке (установка пакетов, компиляция)   |
| `CMD <команда>`        | Команда по умолчанию при запуске контейнера                    |
| `EXPOSE <порт>`        | Документирует, какой порт будет открыт                         |
| `ENV <переменная>`     | Задаёт переменную окружения                                    |
| `WORKDIR <директория>` | Меняет рабочую директорию внутри контейнера                    |
| `USER <пользователь>`  | Указывает пользователя от которого будет стартовать приложение |
## FROM
Всегда первая инструкция. Указывает базовый образ.
``` bash
FROM ubuntu:24.04
FROM python:3.12
FROM nginx:latest
```


## COPY 
Копирует файлы с хоста в образ.
``` bash
COPY app.py /app/
COPY . .
```
Первый `.`:
> текущая директория на хосте

Второй `.`:
> текущая директория внутри контейнера

## RUN
Выполнить команду во время сборки образа.
``` bash
RUN apt update
RUN apt update && apt install -y curl
```

## CMD
Команда при запуске контейнера.
``` bash
CMD ["python", "app.py"]
```

## EXPOSE
Показывает какой порт использует приложение.
``` bash
EXPOSE 80
EXPOSE 5000
```

## ENV
Переменные окружения. 
``` bash
ENV APP_ENV=production
```
Теперь внутри контейнера:
``` bash
echo $APP_ENV # production
```

## WORKDIR
Устанавливает рабочую директорию.
``` bash
WORKDIR /app # Теперь все команды будут выполняться из /app.
```

## USER
Запуск приложения не от root.
``` bash
RUN useradd appuser  
USER appuser
```

## ARG
Переменная только для сборки.
``` bash
ARG VERSION=1.0
```
При сборке:
> `docker build --build-arg VERSION=2.0 .`

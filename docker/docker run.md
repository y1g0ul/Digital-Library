Команда для запуска [[container]] из [[image]] в [[docker]].
``` shell
docker run [OPTIONS] IMAGE [COMMAND] [ARG...]
```
- `--name <name>` - прописать имя контейнеру
- `-i` - интерактивный режим 
- `-t` - создает терминальное окружение
-  `--rm` - контейнер автоматически удаляется после того как он остановится или завершит свою работу
- `-d` - запускает образ отвязывая его от всех стандартных потоков
- `-p` - связывание порта [[container]] с портом хоста (`<порт_хоста>:<порт_контейнера>`)
- `-P` - автоматический проброс на рандомные порты
- `-v` - привязать и смонтировать [[docker volume]]
Запуская снова один и тот же [[image]] создаются все новые и новые [[container]]s. Если [[image]] нет локально на ПК то он будет скачан с [[docker]] hub.
